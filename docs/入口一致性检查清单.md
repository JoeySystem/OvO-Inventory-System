# 入口一致性检查清单

更新时间：2026-03-27

## 目的

本清单用于核对系统界面中的按钮、卡片、导航入口与实际功能落点是否一致，避免出现：

- 文案描述超前于真实能力
- 用户点击后进入错误页面
- 页面存在说明，但并无对应独立入口
- 已实现能力藏在统一页或工单流程中，导致用户误判

## 检查结论

当前系统的入口一致性整体可用：

- 主导航、二级导航、首页快捷入口均有真实落点
- 没发现显式跳转到不存在路由的入口
- 没发现大面积“点 A 去 B”的路径级错误
- 主要问题不在“死入口”，而在“部分说明曾经超前于真实落点”

本轮已按真实实现状态统一为：

- 当前可用
- 已实现但非独立入口
- 规划中

## 二、无落点

本轮未发现显式跳转到不存在路由的入口。

也就是说，当前没有发现下列问题：

- 卡片点击后进入不存在页面
- 菜单点击后出现空路由
- 显式按钮跳到未注册路径

## 三、落点错误

本轮未发现明显的路径级落点错误。

也就是说，当前没有发现：

- 点报表去系统
- 点生产去物料
- 点一致性治理去空页

## 四、已实现但非独立入口

以下业务能力当前已经真实存在，但部分录入动作仍由统一执行页、工单流程或详情页承接：

- 采购收货单
- 完工入库单
- 退料回库单
- 生产领料单
- 库存调整单

当前承接方式：

- 由 `收货入库执行页` 统一承接

#### 2. 完工入库单

当前状态：已具备独立查询页，单据由生产工单完工流程自动生成

当前承接方式：

- 可在工单详情中查看

#### 3. 退料回库单

当前承接方式：

- 由生产工单退料 / 取消流程自动生成
- 可在工单详情中查看

#### 4. 生产领料单

当前状态：已具备独立查询页，单据由生产工单开始执行时生成

当前承接方式：

- 由生产工单开始执行时生成
- 可在工单详情中查看

#### 5. 库存调整单

当前状态：已具备独立查询页，录入仍由盘点调整执行页统一承接

- 由盘点调整执行页统一承接

## 五、规划中

以下功能当前仍属于规划中，还未拆成正式独立入口：

- 车间补料与回仓独立页
- 权限配置页

## 六、当前风险点

当前入口一致性风险主要集中在语义层，而不是路径层：

1. 已实现但非独立入口的业务，容易被用户误认为已经有专门页面
2. 如果后续页面说明继续沿用“阶段性拆分”话术，可能再次造成误解
3. 工单内承接的单据能力，如果没有明确说明“查看位置”，用户会误判为系统缺功能

## 七、后续建议

### P0

- 所有新入口上线前，必须同步标注其真实状态：
  - 当前可用
  - 已实现但非独立入口
  - 规划中

### P1

- 当前主线已完成为“独立查询页 + 统一录入承接”形态，如后续继续演进，可再评估是否拆独立录入页：
  - 生产领料单
  - 完工入库单
  - 退料回库单
  - 采购收货单
  - 库存调整单

### P1

- 每次新增卡片或按钮时，做一次“文案与落点一致性检查”

### P2

- 建立版本化入口台账，和功能开发清单一起维护，避免入口文案滞后

## 八、验收口径

可以视为“入口一致性达标”的条件：

1. 所有显式卡片和按钮都有真实落点
2. 没有跳转到错误页面的路径级问题
3. 所有非独立入口都被明确标识
4. 页面说明与当前真实实现状态一致
5. 新功能上线时同步完成入口说明校对
