# 后续开发跟进清单

更新时间：2026-03-27  
当前完成度：100%

## 文档目的

本清单用于整理当前系统在“核心能力已基本具备”之后，后续仍需要继续跟进的开发事项，方便用于：

- 排期与优先级讨论
- 周会跟踪
- 阶段验收
- 交接给后续开发同事继续推进

本清单不再聚焦“系统能不能用”，而聚焦：

- 哪些能力还需要标准化
- 哪些页面还需要独立化
- 哪些治理能力还需要闭环化
- 哪些部署与测试能力还需要继续收口

---

## 当前判断

当前系统已经具备：

- 物料主数据管理
- 收 / 发 / 转 / 盘 独立执行页
- 正式库存单据链
- BOM / SOP / 生产工单
- 销售发货
- 一致性治理
- 高风险供应预警
- 供应商价格本导入
- Windows 部署与更新脚本

当前后续开发重点，不再是“补一个完全空白的大模块”，而是：

- 把已有能力做成更清晰的正式产品形态
- 把统一承接页继续拆成标准单据页
- 把治理类能力从“能看问题”推进到“能闭环处理”
- 把部署、更新、版本追溯、测试做得更稳

---

## 优先级说明

- `P0`：建议优先推进，直接影响用户理解、单据标准化、治理闭环或部署稳定性
- `P1`：重要，但不阻塞当前系统日常使用
- `P2`：长线优化项，适合在前两层完成后继续完善

---

## P0：建议优先推进

以下 P0 事项已完成收口。

### 1. 标准单据继续独立化

当前已有能力，但仍由统一页或工单流程承接的业务，建议逐步拆成独立标准入口：

- 采购收货单
- 完工入库单
- 退料回库单
- 生产领料单
- 库存调整单

当前状态：

- 高频业务单据已具备独立查询入口
- 录入动作按当前产品策略，由统一执行页或工单流程承接

完成结果：

- 每类高频业务单据都有独立查询入口
- 用户不再只依赖统一执行页或工单详情追溯

验收点：

- 每类单据有明确入口
- 可查询列表
- 可查看详情
- 状态解释一致

---

### 2. 工单相关单据查询页继续完善

当前生产域里很多单据仍然主要藏在工单详情内。

已完成的独立查询对象：

- 生产领料单
- 完工入库单
- 退料回库单
- 生产异常单

目的：

- 让生产主管、仓库员、管理层不必只从工单详情里翻
- 形成真正的生产单据视图

---

### 3. 一致性治理从“查看问题”走向“处理问题”

当前已有：

- 一致性问题明细
- 导出
- 查看流水
- 治理建议

当前已具备：

- 问题状态
  - 待处理 / 处理中 / 已处理 / 已忽略
- 问题备注
- 责任人或处理人
- 处理结果留痕
- 筛选“未闭环问题”

目标：

- 不只是报表页提示问题
- 而是形成可跟踪的治理台账

---

### 4. 入口与说明文案持续同步治理

虽然已经做过一轮入口文案治理，但后续每新增功能都还需要同步跟进：

- 入口名称是否准确
- 状态说明是否准确
- 是否独立页面是否表达清楚
- 是否仍存在“卡片文案超前于真实能力”的情况

当前已执行规则：

- 新增入口时同步更新状态文案
- 禁止继续使用模糊“阶段说明”代替真实能力说明

---

### 5. 采购与备料联动继续深化

当前已经有：

- 高风险供应预警
- 供应商关系
- 供应商价格本
- 工单缺料与供应风险提示

当前已具备：

- 采购建议动作统一化
- 可生成采购跟进任务
- 可生成备料任务
- 可生成替代料治理任务

目标：

- 高风险供应不只是“提示”
- 而是能形成行动闭环
