# 历史采购单据查询开发方案

## 1. 目标定位

本方案用于把管家婆等外部系统导出的“采购单据查询”报表导入 OvO Inventory System，形成一套**历史采购记录查询能力**。

这套能力的定位是：

- 仅用于历史采购记录查询
- 不参与当前库存计算
- 不参与正式库存单据状态流转
- 不写入 `stock_documents / stock_document_items / stock_movements`

## 2. 适用文件

当前目标文件结构为：

- 标题与筛选条件位于前 1-7 行
- 第 8 行为表头
- 第 9 行起为数据

字段包括：

- `录单时间`
- `单据编号`
- `供货单位`
- `入库仓库`
- `数量`
- `采购金额`
- `已付款金额`
- `待付款金额`

## 3. 实现边界

### 做

- 历史采购记录导入 `preview / commit`
- 历史采购记录独立查询页
- 按时间、供应商、单据编号、仓库筛选
- 查看单条历史采购记录详情

### 不做

- 不还原采购单明细行
- 不生成正式采购收货单
- 不参与库存结转和库存流水
- 不参与正式单据状态流转

## 4. 数据模型

新增表：`purchase_records`

字段：

- `id`
- `order_date`
- `order_no`
- `supplier_name`
- `warehouse_name`
- `quantity`
- `amount`
- `paid_amount`
- `unpaid_amount`
- `source_file`
- `raw_source`
- `created_by`
- `created_at`
- `updated_at`

说明：

- `order_no` 作为唯一键，用于二次导入时更新
- `raw_source` 保留原始行数据，方便后续追溯

## 5. 后端接口

### 5.1 导入预览

- `POST /api/purchase-records/import/preview`

能力：

- 自动识别表头
- 跳过表头前的筛选条件区域
- 判断是新增还是更新
- 返回预览摘要与错误信息

### 5.2 导入提交

- `POST /api/purchase-records/import/commit`

能力：

- 基于 `previewToken` 提交
- 按 `order_no` 做新增或更新

### 5.3 列表查询

- `GET /api/purchase-records`

支持：

- 开始日期
- 结束日期
- 供应商
- 单据编号
- 入库仓库
- 分页

### 5.4 详情查询

- `GET /api/purchase-records/:id`

## 6. 前端页面

新增页面：

- `历史采购记录`

建议入口位置：

- `入库` 域二级导航
- `入库` 工作台卡片

页面内容：

- 导入按钮
- 总记录数 / 总采购金额 / 总待付款金额 / 供应商数
- 筛选条件
- 列表表格
- 右侧抽屉式详情

## 7. 用户提示

页面需明确提示：

- `该页面为历史采购记录查询，不参与当前库存计算`
- `该记录不属于正式库存单据链`

## 8. 验收标准

满足以下条件即可视为完成：

1. Excel 可成功 preview
2. Excel 可成功 commit
3. 记录数与原表一致
4. 可按时间、供应商、单据号筛选
5. 可查看详情
6. 页面文案明确说明“仅供历史查询”

## 9. 当前落地顺序

1. 建表与迁移
2. 导入接口
3. 查询接口
4. 前端页面与入口
5. 用真实 Excel 验证导入
